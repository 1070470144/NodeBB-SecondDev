# Quickstart: 剧本（JSON）功能

## 前置条件

- NodeBB 运行正常（Node.js >= 20）
- 管理员账号可登录 ACP
- 已切换到分支：`002-script-management`

## 快速开始（验收路径）

### 1) 用户上传剧本

1. 用普通用户登录
2. 打开“上传剧本”入口（或直接访问 `/scripts/upload`）
3. 选择一个有效 JSON 文件，并填写标题/描述（必填）
4. 提交后确认：
   - 上传成功提示
   - 在“我的剧本”列表可看到该剧本

### 2) 发布到公开页签并在论坛中浏览

1. 将该剧本可见性设置为 `public`（若默认不是公开）
2. 打开 `/scripts`（论坛剧本页签）
3. 验证：
   - 可看到公开剧本列表
   - 搜索关键词（标题/描述）可过滤结果
   - 可按热度（点赞+收藏）/下载量排序

### 3) 点赞、收藏与“我的收藏”

1. 登录用户在某个公开剧本上执行“点赞”
2. 再执行“收藏”
3. 打开 `/scripts/favorites`
4. 验证：
   - 收藏列表中出现该剧本
   - 取消收藏后从列表消失

### 4) 下载量统计

1. 登录用户对某剧本执行一次下载
2. 重复下载同一剧本
3. 验证：
   - 下载量“同一用户终身最多计 1 次”

### 5) 管理端治理

1. 管理员登录 ACP，进入 `/admin/scripts`
2. 对某个剧本执行“下架/封禁”
3. 回到用户侧验证：
   - 剧本不可正常访问/使用（给出可理解提示）
4. 在 ACP 执行“恢复”
5. 验证恢复生效，并可看到审计记录

## 示例 JSON（用于上传）

```json
{
  "name": "demo-script",
  "version": 1,
  "scenes": [
    { "id": "s1", "title": "Opening", "lines": ["Hello", "World"] }
  ]
}
```
